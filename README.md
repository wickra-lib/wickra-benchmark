# wickra-benchmark

> A reproducible, golden-verified benchmark suite for quant backtests: a curated
> collection of `(strategy, dataset, expected report, hash)` cases that anyone
> can recompute in ten languages and confirm byte-for-byte.

`wickra-benchmark` is the "ImageNet for trading-strategy reproducibility". It is
not a new backtest engine ŌĆö it is the curated, hash-pinned **suite** you verify
your engine against. Each case pins a `wickra-backtest` `StrategySpec`, a
deterministic candle dataset, the expected `BacktestReport`, and the `blake3`
hash of that report's canonical form. Re-running a case recomputes the report
with the pinned engine and reports two independent booleans: `passed` (the
recomputed report is byte-identical to the expected one) and `hash_match` (its
canonical hash equals the pinned `expected_hash`).

The core (`benchmark-core`) is exposed as a JSON-over-C-ABI data API in ten
languages, mirroring the Wickra tiering: a Rust core with native Python, Node.js
and WASM bindings plus a C ABI hub for C, C++, C#, Go, Java and R. Every binding
returns the core's canonical JSON verbatim, so a suite report is byte-identical
in every language and between the parallel (rayon) and sequential (WASM) runners.

## The registry

- `datasets/` ŌĆö curated, deterministic candle CSVs (`ts,open,high,low,close,volume`),
  each generated by a documented formula and pinned by a manifest hash.
- `cases/` ŌĆö one `BenchmarkCase` JSON per case, plus `cases/suite.json` listing
  the full curated suite. Small enough to recompute by hand, broad enough to
  prove engine reproducibility.

## Use in any language

The core drives a single `command_json` envelope (`run_case`, `run_suite`,
`list_cases`, `version`); every binding passes the same string through verbatim.

## Building

Rust `1.86+`. `cargo build` builds the workspace; `cargo test --workspace
--all-features` runs the suite. Per-binding build steps live in each
`bindings/<lang>/README.md`.
